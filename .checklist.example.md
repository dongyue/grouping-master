# 开发流程检查清单

> 从 `.checklist.example.md` 复制，逐项打勾，未打勾的项不得进入下一步。
> 「→ 用户确认」的项，必须用户回复确认后方可打勾。
> 该文件不纳入版本控制。

**Feature 分支**: `feature/xxx`

- [ ] 0. 需求讨论完成，用户认为 OK → 用户确认
- [ ] 1. 已从 master 拉出 feature 分支
- [ ] 2. `docs/requirements.md` 已更新 → 用户确认
- [ ] 3. `docs/design.md` 已更新 → 用户确认
- [ ] 4. 代码已编写
- [ ] 5. 单测通过（若有算法改动）：`cd backend && source venv/bin/activate && python3 -m pytest tests/ -v`
- [ ] 6. 用户人工测试验收通过 → 用户确认
- [ ] 7. `docs/requirements.md`、`docs/design.md`、`docs/codebase-map.md` 与源代码一致 → 用户确认
- [ ] 8. commit 所有改动，合入 master
