# 分组大师 - 需求文档

## 1. 成员管理

### 1.1 用户注册
- 用户通过账号名、昵称、密码注册账号
- 可选填写备用邮箱、上传头像
- 账号名规则：3-50 位，仅允许字母、数字、下划线
- 密码规则：至少 8 位
- 账号名和邮箱（如填写）全局唯一
- 注册成功后自动登录

### 1.2 用户登录
- 使用账号名 + 密码登录
- 支持持久登录：关闭浏览器后 30 天内无需重新输入（Cookie Session）
- 登录失败显示明确错误提示

### 1.3 用户登出
- 用户可主动退出登录，清除服务端 Session

### 1.4 修改密码
- 登录用户可修改密码，需验证旧密码
- 新密码至少 8 位

### 1.5 忘记密码
- 用户输入备用邮箱，系统发送重置密码链接
- 重置链接 30 分钟内有效
- 未注册邮箱不暴露用户是否存在（统一返回提示）
- 重置后销毁该用户所有 Session，需重新登录

### 1.6 个人资料
- 查看和修改昵称
- 上传头像（支持 JPG/PNG/GIF，最大 2MB）

### 1.7 路由鉴权
- 未登录访问需登录页面 → 重定向到登录页
- 已登录访问登录/注册页 → 重定向到首页
- 页面刷新后自动恢复登录状态

### 1.8 注销账号
- 用户可在设置页注销账号
- 操作前弹出确认提示，防止误操作
- 注销后删除用户数据、头像文件及关联的 session 和重置令牌

## 2. 活动管理

### 2.1 创建活动
- 登录用户在首页可创建活动
- 活动信息：标题（必填）、描述（可选）
- 标题不能为空
- 创建表单提供「同时加入活动」复选框，默认勾选
- 勾选时，创建者创建活动后自动成为活动成员
- 不勾选时，创建者仅创建活动但不加入，后续可手动操作加入
- 创建时系统自动生成唯一 slug 作为活动公网标识，替代自增 ID 暴露在 URL 中

### 2.2 加入活动
- 用户可通过活动链接进入活动详情页，点击「加入活动」按钮加入该活动
- 已加入的用户在详情页不显示「加入活动」按钮
- 加入成功后，该活动出现在用户首页「我加入的活动」列表中

### 2.3 活动列表
- 登录后的首页展示两个列表：「我创建的活动」和「我加入的活动」
- 「我创建的活动」列出当前用户创建的所有活动
- 「我加入的活动」列出当前用户加入的所有活动（含自己创建并加入的）
- 均按创建时间倒序排列
- 每条显示活动标题、描述（截断显示）

### 2.4 活动详情页
- 每个活动有独立的详情页，URL 为 `/activities/:slug`
- 页面展示活动的完整信息：标题、描述、创建者昵称、创建时间
- 需要登录才能查看（系统内任意注册用户均可查看）
- 页面展示「已加入的成员」列表，每项显示成员昵称和头像，按加入时间排列；列表标题显示参与总人数（如「已加入的成员 8 人」）
- 成员列表实时反映当前已加入状态：用户加入后刷新列表即可看到自己
- 从首页活动列表点击活动条目可跳转到详情页
- 页面操作按钮详见 2.5 操作按钮

### 2.5 活动详情页操作按钮

- 操作按钮处于活动标题下方、成员列表上方，按使用频率从左到右排列：
  - 加入活动 / 开始分组（核心操作）
  - 分享链接
  - 「更多」下拉菜单，内含退出活动、解除分组（仅创建者）、编辑活动（仅创建者）、删除活动（仅创建者）
- 按钮颜色：加入活动、开始分组使用主色按钮；分享链接使用次色按钮；退出活动、解除分组使用警告色按钮（在「更多」菜单内）；编辑活动使用次色按钮（在「更多」菜单内）；删除活动使用红色危险按钮（在「更多」菜单内）
- 用户通过浏览器返回或顶栏导航回到首页

### 2.6 删除活动
- 活动创建者可在活动详情页删除自己创建的活动
- 非创建者无删除入口
- 删除前弹出确认提示，防止误操作
- 删除活动后，该活动的所有成员关系一并清除
- 删除成功后跳转到首页，提示「活动已删除」

### 2.7 退出活动
- 活动成员可在详情页「更多」菜单中退出该活动
- 仅已加入的成员可退出
- 退出前弹出确认提示，防止误操作
- 退出成功后，该活动从「我加入的活动」列表中消失

### 2.8 编辑活动
- 活动创建者可在详情页「更多」菜单中编辑活动，前往编辑页面
- 编辑页面预填当前标题和描述
- 可修改标题（必填）和描述（可选），保存后返回详情页
- 可放弃编辑，不保存直接返回详情页
- 编辑页面入口仅创建者可见
- 非创建者拼凑编辑页 URL 访问时，重定向到活动详情页

### 2.9 踢出成员
- 活动创建者可在详情页成员列表中将某成员踢出活动
- 成员列表标题旁设「管理成员」切换开关，仅创建者可见，默认关闭
- 打开开关后，每条成员旁显示「踢出」按钮；关闭后隐藏
- 踢出前弹出确认提示，防止误操作
- 创建者不能踢出自己（自己应使用「退出活动」功能）

### 2.10 分组

- 活动创建者在活动详情页点击「开始分组」按钮触发自动分组
- 分组规则：成员列表随机打乱，按每组 2 人分配，若最后剩余 1 人则单独成组
- 分组后，活动详情页中原来的成员平铺列表替换为按组展示的列表，每组显示成员昵称和头像；成员列表标题额外显示总组数（如「已加入的成员 8 人，共 4 组」）
- 分组后，成员操作冻结：「加入活动」按钮变为禁用状态，「更多」菜单中「退出活动」变为禁用状态，踢出按钮不再显示；「开始分组」按钮隐藏，「更多」菜单中显示「解除分组」
- 分组后，「开始分组」按钮隐藏，「更多」菜单中显示「解除分组」；点击后清除分组数据，活动恢复未分组状态，成员操作解冻

## 3. 安全防护

### 3.1 接口速率限制
- 对登录（`POST /api/auth/login`）、注册（`POST /api/auth/register`）、忘记密码（`POST /api/auth/forgot-password`）三个接口实施 IP 级别速率限制
- 限制规则：同一 IP 对上述单个接口在 1 分钟内最多请求 5 次
- 超限后返回 429 Too Many Requests，并提示用户稍后再试
- 限流数据仅存储在应用内存中，服务重启后清零

### 3.2 头像文件魔术头校验
- 上传头像时，除检查 HTTP Content-Type 头外，校验文件内容开头的魔术头字节（Magic Bytes），防止攻击者伪造 MIME 类型上传恶意文件
- 允许的文件类型及对应魔术头：
  - JPEG：`FF D8 FF`
  - PNG：`89 50 4E 47`
  - GIF：`47 49 46 38`
- 魔术头校验失败返回 400 错误，提示「文件类型与扩展名不符」
