# 分组大师 - 需求文档

## 1. 成员管理

### 1.1 用户注册
- 用户通过账号名、昵称、密码注册账号
- 可选填写备用邮箱、上传头像
- 账号名规则：3-50 位，仅允许字母、数字、下划线
- 密码规则：至少 8 位
- 账号名和邮箱（如填写）全局唯一
- 注册成功后自动登录

### 1.2 用户登录
- 使用账号名 + 密码登录
- 支持持久登录：关闭浏览器后 30 天内无需重新输入（Cookie Session）
- 登录失败显示明确错误提示

### 1.3 用户登出
- 用户可主动退出登录，清除服务端 Session

### 1.4 修改密码
- 登录用户可修改密码，需验证旧密码
- 新密码至少 8 位

### 1.5 忘记密码
- 用户输入备用邮箱，系统发送重置密码链接
- 重置链接 30 分钟内有效
- 未注册邮箱不暴露用户是否存在（统一返回提示）
- 重置后销毁该用户所有 Session，需重新登录

### 1.6 个人资料
- 查看和修改昵称
- 上传头像（支持 JPG/PNG/GIF，最大 2MB）

### 1.7 路由鉴权
- 未登录访问需登录页面 → 重定向到登录页
- 已登录访问登录/注册页 → 重定向到首页
- 页面刷新后自动恢复登录状态

### 1.8 注销账号
- 用户可在设置页注销账号
- 操作前弹出确认提示，防止误操作
- 注销后删除用户数据、头像文件及关联的 session 和重置令牌

## 2. 活动管理

### 2.1 创建活动
- 登录用户在首页可创建活动
- 活动信息：标题（必填）、描述（可选）
- 标题不能为空
- 创建表单提供「同时加入活动」复选框，默认勾选
- 勾选时，创建者创建活动后自动成为活动成员
- 不勾选时，创建者仅创建活动但不加入，后续可手动操作加入
- 创建成功后活动归属于当前用户

### 2.2 加入活动
- 用户可通过活动链接进入活动详情页，点击「加入活动」按钮加入该活动
- 已加入的用户在详情页不显示「加入活动」按钮
- 加入成功后，该活动出现在用户首页「我加入的活动」列表中

### 2.3 活动列表
- 登录后的首页展示两个列表：「我创建的活动」和「我加入的活动」
- 「我创建的活动」列出当前用户创建的所有活动
- 「我加入的活动」列出当前用户加入的所有活动（含自己创建并加入的）
- 均按创建时间倒序排列
- 每条显示活动标题、描述（截断显示）

### 2.4 活动详情页
- 每个活动有独立的详情页，URL 为 `/activities/:id`
- 页面展示活动的完整信息：标题、描述、创建者昵称、创建时间
- 需要登录才能查看（系统内任意注册用户均可查看）
- 页面提供分享按钮，点击可复制链接到剪贴板
- 页面提供「加入活动」按钮，仅当用户尚未加入该活动时显示（与是否为创建者无关）
- 页面展示「已加入的成员」列表，每项显示成员昵称和头像，按加入时间排列
- 成员列表实时反映当前已加入状态：用户加入后刷新列表即可看到自己
- 从首页活动列表点击活动条目可跳转到详情页

### 2.5 删除活动
- 活动创建者可在活动详情页删除自己创建的活动
- 非创建者不显示删除操作入口
- 删除前弹出确认提示，防止误操作
- 删除活动后，该活动的所有成员关系一并清除
- 删除成功后跳转到首页，提示「活动已删除」
