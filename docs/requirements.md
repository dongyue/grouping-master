# 分组大师 - 需求文档

> 需求编号前缀 `REQ-`，避免与设计文档编号冲突。

## 1. 成员管理

### 1.1 用户注册
- 用户通过账号名、昵称、密码注册账号
- 可选填写备用邮箱、上传头像
- 账号名规则：3-50 位，仅允许字母、数字、下划线
- 密码规则：至少 8 位；通过 `.env` 中 `REQUIRE_PASSWORD` 可关闭密码要求（调试用，默认 `true` 必须设置）
- 账号名和邮箱（如填写）全局唯一
- 注册成功后自动登录
- 提交前校验两次输入密码是否一致，不一致时当场提示

### 1.2 用户登录
- 使用账号名 + 密码登录
- 支持持久登录：关闭浏览器后 30 天内无需重新输入（Cookie Session）
- 登录失败显示明确错误提示

### 1.3 用户登出
- 用户可主动退出登录，清除服务端 Session

### 1.4 修改密码
- 登录用户可修改密码，需验证旧密码
- 新密码至少 8 位
- 提供确认新密码输入框，防止输入错误
- 修改成功后提示自动消失

### 1.5 忘记密码
- 用户输入备用邮箱，系统发送重置密码链接
- 重置链接 30 分钟内有效
- 重置密码页面提供确认密码输入框
- 未注册邮箱不暴露用户是否存在（统一返回提示）
- 重置后销毁该用户所有 Session，需重新登录

### 1.6 个人资料
- 查看和修改昵称（个人设置页可直接改；在活动中修改也会同步回个人设置，下次加入新活动时自动预填）
- 上传头像（支持 JPG/PNG/GIF，最大 2MB）

### 1.7 注销账号
- 用户可在设置页注销账号
- 操作前弹出确认提示，防止误操作
- 注销后删除用户数据、头像文件及关联的 session 和重置令牌，并自动退出登录状态

### 1.8 个人属性
- 用户在加入活动或编辑个人信息时填写的属性名值对，自动同步存储到个人设置中
- 个人设置页展示用户的所有属性值列表，每个属性名只能有一个值（属性名不可重复），支持添加、修改、删除
- 下次加入活动或编辑个人信息时，已保存的同名属性值若在该活动的枚举值中，自动预填
- 删除活动时不影响保存下来的个人属性值；删除个人属性值时不影响各活动中的已填属性

## 2. 活动管理

### 2.1 创建活动
- 登录用户在首页「我创建的活动」标题旁点击「创建活动」按钮，进入独立的创建活动页面
- 活动信息：标题（必填）、描述（可选）
- 标题不能为空
- 创建表单提供「我作为创建者也要参加」复选框，默认勾选
- 不勾选时，创建者仅创建活动但不加入，后续可手动操作加入
- 勾选时，创建活动后弹出个人信息表单，用户确认昵称和属性值后成为成员（见 2.2）

### 2.2 加入活动
- 用户可通过活动链接进入活动详情页，点击「加入活动」按钮加入该活动
- 已加入的用户在详情页不显示「加入活动」按钮
- 加入成功后，该活动出现在用户首页「我加入的活动」列表中
- 点击「加入活动」后始终弹出个人信息表单，包含昵称字段（预设用户当前昵称，可修改，旁注「此处有时宜填写真实姓名」）、该活动约束规则所对应的属性选择（如有，详见 3.2），以及成员偏好选择（若活动已开启，详见 3.3）
- 提交后成为成员；活动中修改的信息（昵称、属性值）自动同步回个人设置

### 2.3 活动列表
- 登录后的首页展示两个列表：「我创建的活动」和「我加入的活动」
- 「我创建的活动」标题旁有「创建活动」按钮，点击前往独立的创建活动页面
- 「我创建的活动」列出当前用户创建的所有活动
- 「我加入的活动」列出当前用户加入的所有活动（含自己创建并加入的）
- 均按创建时间倒序排列
- 每条显示活动标题、描述（截断显示）

### 2.4 活动详情页
- 每个活动有独立的详情页，URL 为 `/activities/:slug`
- 页面展示活动的完整信息：标题、描述、创建者昵称、创建时间
- 需要登录才能查看（系统内任意注册用户均可查看）
- 页面分为「分组规则」和「成员情况」两个区域，各有独立标题
- 「分组规则」标题下展示分组方式、组内多样性限定规则（如有），以及已开启的成员偏好设定（如有），详见 3.5
- 「成员情况」标题：未分组时显示「成员情况」，已分组时显示「成员与分组情况」；标题右侧依次为排序切换下拉和「管理成员」按钮
- 标题下一行显示概述（如「成员 8 人」或「成员 8 人，分为 3 组，另有 2 人落单」，落单人数为 0 时省略落单部分）
- 成员列表每项显示成员昵称、头像及各属性值（若有）；已分组且非按分组排序时额外显示所在组号或「落单」
- 排序切换下拉提供以下选项：按加入时间（默认）、按昵称，以及按各约束属性的值（如有约束规则）；已分组时额外增加「按分组」选项
- 切换立即生效
- 按加入时间或按昵称排序时，为平铺列表
- 按某约束属性的值排序时，按该属性取值分类展示，每个取值一个分类卡片，标题为属性值及人数；未填写该属性的成员归入「未填写」分类
- 按分组排序时，按组展示，每组显示成员昵称和头像；组列表下方展示「落单」区域列出未分配成员
- 分类展示或按组展示时，当前用户所在分类或组整体高亮；若自己在落单列表中，同样高亮
- 成员列表中，当前登录用户的条目以高亮背景或边框突出显示，便于定位自己
- 成员属性值在成员条目中展示，分组和未分组视图下均可见
- 成员列表实时反映当前已加入状态：用户加入后刷新列表即可看到自己
- 从首页活动列表点击活动条目可跳转到详情页
- 页面操作按钮详见 2.5 操作按钮

### 2.5 活动详情页操作按钮

- 操作按钮处于活动标题下方、成员列表上方，按使用频率从左到右排列：
  - 加入活动 / 自动分组（核心操作）
  - 手动调整（仅创建者，已分组时）
  - 分享链接
  - 「更多」下拉菜单，内含退出活动（已加入成员）、重新分组（仅创建者，已分组时）、解除分组（仅创建者，已分组时）、编辑活动（仅创建者）、删除活动（仅创建者）
- 按钮颜色：加入活动、自动分组使用主色按钮；手动调整使用次色按钮，点击后进入手动调整状态，按钮文案变为「完成调整」；分享链接使用次色按钮；退出活动、解除分组使用警告色按钮（在「更多」菜单内）；编辑活动使用次色按钮（在「更多」菜单内）；删除活动使用红色危险按钮（在「更多」菜单内）
- 用户通过浏览器返回或顶栏导航回到首页

### 2.6 删除活动
- 活动创建者可在活动详情页删除自己创建的活动
- 非创建者无删除入口
- 删除前弹出确认提示，防止误操作
- 删除活动后，该活动的所有成员关系一并清除
- 删除成功后跳转到首页，提示「活动已删除」

### 2.7 退出活动
- 活动成员可在详情页「更多」菜单中退出该活动
- 仅已加入的成员可退出
- 退出前弹出确认提示，防止误操作
- 退出成功后，该活动从「我加入的活动」列表中消失
- 若活动已分组，退出时同步从所在组中移除该成员

### 2.8 编辑活动
- 活动创建者可在详情页「更多」菜单中编辑活动，前往编辑页面
- 编辑页面预填当前标题和描述
- 可修改标题（必填）和描述（可选），保存后返回详情页
- 可放弃编辑，不保存直接返回详情页（非浏览器历史上一页）
- 编辑页面入口仅创建者可见
- 非创建者拼凑编辑页 URL 访问时，重定向到活动详情页

### 2.9 踢出成员
- 活动创建者可在详情页成员列表中将某成员踢出活动
- 成员列表标题旁设「管理成员」切换开关，仅创建者可见，默认关闭
- 打开开关后，每条成员旁显示「踢出」按钮，按钮文案变为「完成管理」；关闭后隐藏
- 踢出前弹出确认提示，防止误操作
- 创建者不能踢出自己（自己应使用「退出活动」功能）
- 若活动已分组，踢出时同步从所在组中移除该成员

### 2.10 操作日志
- 系统在活动发生关键操作时自动记录日志
- 记录的操作类型包括：创建活动、编辑活动、加入活动、退出活动、踢出成员、执行分组（含重新分组）、解除分组、编辑个人信息
- 每条日志包含：操作人昵称、操作内容描述、操作时间戳（精确到秒）
- 编辑活动时记录变更内容（标题、描述的改动）
- 分组类操作（执行分组 / 重新分组）除基本日志信息外，额外以结构化数据记录分组快照：
  - 分组策略、分组参数、约束规则
  - 分组时活动所有成员及其属性值
  - 分组算法参数（随机种子、成员打乱前的顺序）
  - 分组结果（各组编号及成员分配、落单成员列表）
  - 成员偏好统计（若有人设置了偏好）
- 活动创建者可在活动详情页「更多」菜单中点击「查看日志」，跳转到独立的日志页面 `/activities/:slug/logs`
- 日志列表按时间倒序展示，普通日志展示操作描述和时间；分组日志可展开查看分组快照详情
- 日志页面仅活动创建者可访问，非创建者访问时重定向到活动详情页

## 3. 自动分组

### 3.1 分组方式

- 活动创建者在创建活动和编辑活动时可配置分组方式，两个选项：
  - 固定每组人数（默认）：指定每组目标人数，系统自动计算组数。参数为每组人数，默认 2，最小 2
  - 固定总组数：指定目标组数，系统将总人数尽可能平均分配到各组。参数为总组数，默认 2，最小 2，最大不超过当前活动成员总数
- 两种策略下，不能满足整除条件时（有余数），余下的成员归入「落单」，不分配进任何组内
  - 固定每组人数示例：10 人，每组 3 人 → 3 组（各 3 人），1 人落单
  - 固定总组数示例：10 人，分 3 组 → 3 组（各 3 人），1 人落单

### 3.2 组内多样性限定

- 活动创建者在创建活动和编辑活动时可配置组内多样性限定规则，0 条到多条
- 每条规则包含：
  - 属性名：提供预设选项（性别、团队、部门等）供下拉选择，也支持自定义输入
  - 属性值枚举：该属性所有可能取值的列表，如「男」「女」。选择预设属性「性别」时自动填入「男，女」
  - 限定类型：二选一——限定最小值或限定最大值
  - 限定值：正整数。限定最小值时，约束各组该属性不同值数量不少于该值；限定最大值时，约束各组该属性不同值数量不超过该值
- 限定最小值示例：属性名「性别」，值枚举「男」「女」，限定最小值 2 → 每组必须男女都有（如结伴跳舞场景）
- 限定最大值示例：属性名「性别」，值枚举「男」「女」，限定最大值 1 → 每组必须全部同性别（如住宿分配场景）
- 限定值的合法性校验：限定最小值时，限定值介于 2 和枚举值数量之间；限定最大值时，限定值介于 1 和枚举值数量减 1 之间
- 同一活动内，属性名不可重复
- 约束规则作为分组算法的输入，系统在分组时自动满足各组内的多样性限定

### 3.3 成员偏好设置
- 活动创建者在创建/编辑活动时可开启两项独立的成员偏好功能，并设定每人可选人数上限：
  - 「想和谁同组」：开关默认关闭；开启时需设定上限（1-10 之间，默认 1）
  - 「不想和谁同组」：开关默认关闭；开启时需设定上限（1-10 之间，默认 1）
- 两种偏好可单独开启、同时开启或均关闭
- 活动开启某偏好后，成员加入活动（见 2.2）或编辑个人信息（见 3.4）时，表单底部额外显示对应偏好选择区域
- 「想和谁同组」文案为「尽量安排与他/她们同组」（上限为 1 人时显示「尽量安排与他/她同组」）
- 「不想和谁同组」文案为「尽量避免与他/她们同组」（同上）
- 通过多选组件从当前已加入的其他成员中选择目标，成员按昵称排序，每人显示属性值（如「张三（男，企划部）」），可选人数不超过设定的上限
- 选择器底部显示已选总结（如「当前未指定」或「已指定 2（最多 3）：张三、李四」）
- 第一项偏好标题下方注明「此名单系统不会透露给任何人」（两项偏好开启时加「下同」）；弹窗顶部显示引导语「请在本页填写您在本次活动中的个人信息，加入活动后可在编辑页中继续修改完善」
- 成员退出或被踢出时，其偏好记录同步删除（数据库 CASCADE）
- 被选为目标的人若退出/被踢出，系统自动从当前用户的偏好中移除该目标；活动规则变更导致人数上限缩小时，读取时自动截断超限部分

### 3.4 成员个人设置
- 成员在活动中的个人设置包括：昵称、约束属性值（如有）、偏好选择（若活动已开启，见 3.3）
- 约束属性值为必填，必须从活动对应规则定义的允许值中选取；属性值与成员记录关联存储，可作为分组算法依据
- 成员加入后若活动约束规则被修改，可能导致已有属性值不合规（缺少某属性、或值不在当前允许值列表中）。成员列表中对不合规的成员显示警告图标，鼠标悬停时提示具体合规性问题
- 成员可随时修改个人设置，有两个入口：
  - 成员列表中自己条目旁的编辑图标（始终可见）
  - 详情页「更多」菜单中的「编辑我的信息」（始终可见）
- 编辑时弹出与加入时相同的个人信息表单，预填当前昵称、有效属性值，以及已有的偏好选择

### 3.5 详情页展示

- 活动详情页「分组规则」标题下展示分组方式、组内多样性限定规则（如有），以及已开启的成员偏好设定（如有）。分组方式格式如「分组方式：每组 3 人」或「分组方式：共分 3 组」，多样性规则格式如「组内限定：每组至少 2 种性别」，偏好设定格式如「成员偏好：允许成员设置「想和谁同组」，最多 2 人」
- 以上展示对活动所有成员和访客均可见
- 分组后，成员列表支持四种排序方式展示，详见 2.4
- 分组后，详情页操作按钮：未加入用户始终显示「加入活动」；已加入成员始终可退出；「自动分组」按钮（仅创建者）在未分组时显示于主按钮栏，已分组时隐藏，改为「更多」菜单中的「重新分组」；已分组时主按钮栏显示「手动调整」（仅创建者），「更多」菜单中显示「解除分组」

### 3.6 执行与解除分组

- 活动创建者在活动详情页点击「自动分组」按钮触发自动分组；已分组后该按钮隐藏，「更多」菜单中显示「重新分组」
- 执行分组前，系统检查所有成员的属性值是否完整且合法：
  - 若活动定义了约束规则，每个成员须为每条规则中的属性选好值，且该值在当前允许值列表中。属性缺失或值不在允许范围内视为不合格
  - 全部成员合格后方可执行分组；如有不合格成员，阻止分组并返回详细报告（列出每个不合格成员的昵称及其具体问题）
- 重新分组时，先清除已有分组数据，再按当前成员和最新配置重新执行分组
- 分组后新加入的成员归入「落单」
- 点击「解除分组」后清除分组数据，活动恢复未分组状态

### 3.7 分组算法规则

- 分组时各因素的优先级从高到低为：分组方式设置 → 多样性限定规则（硬约束）→ 成员偏好（软约束）→ 随机性
- 核心原则：在满足分组方式参数和多样性硬约束的前提下，使尽可能少的人落单；同等硬约束和落单条件下，尽量满足成员偏好；同等偏好满足度下保持随机性，避免每次分组结果雷同
- 对「固定每组人数」策略：按每组目标人数分组，无论整除与否，能符合约束的组尽量建满；实在无法满足约束的成员归入落单
- 对「固定总组数」策略：尽可能建满目标组数，但如果某些组找不到符合约束的成员组合，宁可少建组，每个已建组须合规，余下成员归入落单
- 多样性限定规则为硬约束，必须满足
- 成员偏好为软约束，在满足硬约束且不增加落单人数的前提下尽量满足

#### 3.7.1 偏好评分规则

- 偏好评分规则用于衡量一对成员之间同组意愿的强弱，分组时尽量将得分高的组合放在同组、得分低的组合分到不同组
- 两个成员之间的偏好关系得分，基于双方各自设置的 want（想同组）和 avoid（不想同组），按以下优先级判定：
  - avoid 优先于 want：只要任一方设置了 avoid，该关系即视为避免同组关系，忽略 want
  - 互相（双方均设 avoid）优先级高于单向（仅一方设 avoid）
  - 互相（双方均设 want）优先级高于单向（仅一方设 want）
- 具体评分：

| 关系 | 得分 |
|------|------|
| 双方互相 avoid | -6 |
| 单方 avoid（对方未设 avoid） | -2 |
| 双方互相 want（且无 avoid） | +3 |
| 单方 want（且无 avoid） | +1 |
| 无偏好 | 0 |

## 4. 手动调整

- 自动分组完成后，活动创建者可在详情页点击「手动调整」按钮进入手动调整状态，按钮文案变为「完成调整」
- 手动调整状态下，成员列表切换为可拖拽调整分组的界面，具体操作方式另文定义
- 点击「完成调整」退出手动调整状态，分组结果恢复为只读展示
- 手动调整状态下不影响其他操作（加入/退出/踢出成员、重新分组、解除分组等均照常可用）

## 5. 安全防护

### 5.1 接口速率限制
- 对登录、注册、忘记密码三个接口实施 IP 级别速率限制
- 限制规则：同一 IP 对上述单个接口在 1 分钟内最多请求 5 次
- 超限后提示用户稍后再试
- 限流数据仅存储在应用内存中，服务重启后清零

### 5.2 头像文件魔术头校验
- 上传头像时，除检查 HTTP Content-Type 头外，校验文件内容开头的魔术头字节（Magic Bytes），防止攻击者伪造 MIME 类型上传恶意文件
- 允许的文件类型及对应魔术头：
  - JPEG：`FF D8 FF`
  - PNG：`89 50 4E 47`
  - GIF：`47 49 46 38`
- 魔术头校验失败返回 400 错误，提示「文件类型与扩展名不符」

## 6. 用户体验

### 6.1 全局样式
- 页面在不同尺寸屏幕下布局正常，无样式冲突导致的错乱
- 加载中状态统一展示加载动画
- 无数据时统一展示空状态提示

### 6.2 页面导航
- 未登录访问需登录页面 → 重定向到登录页
- 已登录访问登录/注册页 → 重定向到首页
- 页面刷新后自动恢复登录状态
- 访问不存在的页面路径时，显示 404 提示页，提供返回首页入口
- 未登录用户访问需登录的页面时，登录后自动回到目标页面

### 6.3 确认对话框
- 危险操作（退出活动、删除活动、踢出成员、解除分组、注销账号）使用样式统一的确认对话框，替代浏览器原生弹窗

### 6.4 日期显示
- 所有页面中日期的显示格式统一为 `YYYY-MM-DD`

### 6.5 设置页帮助说明
- 个人设置页在昵称和属性值区域旁提供帮助说明，解释其与活动中表单的联动关系：
  - 昵称：是参加活动时的默认显示名，在活动中修改会反写回此处
  - 属性值：是各约束属性的个人默认值，参加活动时如有同名属性且值在活动选项内则自动预填，在活动中填写或修改值也会反写回此处
